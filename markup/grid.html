# Grid Layout

Grid 속성 같은 경우는 IE를 지원하지 않습니다. 제가 일한 곳에서는 IE를 지원하기 때문에 실무에 써본 경험은 없지만 최신 속성도 계속 공부 및 예제를 통해 학습하고있습니다.

# 그리드 정의

최상위 부모에 display:grid 또는 display:inline-grid를 정의합니다. 그런 다음 grid-template-columns및 grid-template-rows속성을 사용하여 그리드를 만들 수 있습니다. 이 grid-gap속성을 사용하여 열과 10px의 행 사이에 간격을 만듭니다. 부모의 모든 직계 자식이 이제 그리드 항목이되고 자동 배치 알고리즘이 각 그리드 셀에 하나씩 배치합니다. 필요에 따라 추가 행을 만듭니다.

<div class="wrapper">
    <div class="box a">A</div>
    <div class="box b">B</div>
    <div class="box c">C</div>
    <div class="box d">D</div>
    <div class="box e">E</div>
    <div class="box f">F</div>
</div>

body {
    margin: 40px;
}
.wrapper {
    display: grid;
    grid-template-columns: 100px 100px 100px;
    grid-gap: 10px;
    background-color: #fff;
    color: #444;
}
.box {
    background-color: #444;
    color: #fff;
    border-radius: 5px;
    padding: 20px;
    font-size: 150%;
}

# 명시적 및 암시적 그리드

우리가 사용하는 grid-template-columns와 grid-template-rows 우리는 만들 명시 그리드. 그러나 해당 그리드 외부에 항목을 배치하려고 시도하면 브라우저는 해당 항목을 보유하기 위해 암시 적 그리드 선을 만듭니다.

아래 코드 e에서 그리드 열 라인 4와 5 사이에 배치했지만 이것으로 설명되지 않으므로 grid-template-rows 암시 적 그리드 라인 5가 생성됩니다.

기본적으로 암시 적 선으로 만들어진 암시 적 그리드 트랙의 크기는 자동으로 조정됩니다. 그러나 grid-auto-columns및 grid-auto-rows속성을 사용 하여 트랙 크기를 조정할 수 있습니다. 그리드의 나머지 열 트랙과 일치하도록 자동 트랙의 크기를 100px로 조정했습니다.

    <div class="box a">A</div>
    <div class="box b">B</div>
    <div class="box c">C</div>
    <div class="box d">D</div>
    <div class="box e">E</div>
</div>

body {
    margin: 40px;
}
.wrapper {
    display: grid;
    grid-gap: 10px;
    grid-template-columns: 100px 100px 100px;
    grid-auto-columns: 100px;
    background-color: #fff;
    color: #444;
}
.box {
    background-color: #444;
    color: #fff;
    border-radius: 5px;
    padding: 20px;
    font-size: 150%;
}
.a {
    grid-column: 1 / 3;
    grid-row: 1;
}
.b {
    grid-column: 3 ;
    grid-row: 1 / 3;
}
.c {
    grid-column: 1 ;
    grid-row: 2 ;
}

    grid-row: 1 / 4;

# 그리드 영역 정의

그리드에 이름이 지정된 영역을 만들어 내용을 넣을 수 있습니다. 이를 위해 먼저 grid-area속성을 사용하여 마크 업의 요소를 그리드 영역에 할당 합니다. 그런 다음이 grid-template-areas속성을 사용하여 그리드에서 이러한 요소가 어디에 위치하는지 설명 할 수 있습니다. 영역 이름을 반복하면 해당 영역이 해당 그리드 트랙에 걸쳐 있음을 나타냅니다. 또는 같은 시퀀스를 사용 ....하면 빈 트랙을 나타냅니다. 이 그리드의 첫 번째 셀과 같습니다.

<div class="wrapper">
    <div class="box header">Header</div>
    <div class="box sidebar">Sidebar</div>
    <div class="box content">Content</div>
</div>

body {
    margin: 40px;
}
.sidebar {
    grid-area: sidebar;
}
.content {
    grid-area: content;
}
.header {
    grid-area: header;
}
.wrapper {
    display: grid;
    grid-gap: 10px;
    grid-template-columns: 120px  120px  120px;
    grid-template-areas: "....... header  header" "sidebar content content";
    background-color: #fff;
    color: #444;
}
.box {
    background-color: #444;
    color: #fff;
    border-radius: 5px;
    padding: 20px;
    font-size: 150%;
}
.header {
    background-color: #999;
}

# 꼭 필요하진 않음

마크업에 바닥 글을 추가하고을 정의하고 grid-area이를 사용하여 그리드에 배치 grid-template-areas하면 컨텐츠 열 아래에 있음을 알 수 있습니다. 이 바닥 글에 그리드에 트랙이 있으므로 다른 트랙으로 이동할 수 없으므로 지우기가 필요하지 않습니다.

이 예제는 또한 예제 5에 설명 된 문제를 보여줍니다. 기본적으로 사이드 바의 배경은 Flexbox의 기본값과 비슷한 방식으로 전체 높이로 확장 되어야합니다.

<div class="wrapper">
