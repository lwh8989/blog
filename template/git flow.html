# Git Flow

참 이야기가 많은 도구인데요... 제가 일하던 마크업 파트에서는 형상관리를 사실 제대로 못하고있었습니다. 개발과 마크업싱크는 항상 맞지않았었습니다. 조직개편을 통해서 마크업이 프론트개발팀으로 흡수되면서 형상관리에 중요성을 느꼈고 익숙해지는 기간동안 저 뿐만아니라 팀원 및 협력사의 싱크가 맞지않아서 꼬이는 현상이 자주 발생했습니다.

  • 우아한형제들 git flow

프론트를 하는 친한 동료가 이 글을 추천해줬을때는 이해가 안가던 부분이 사고를 몇번 쳐보니까 이해가 갔습니다. 역시...삽지를 해봐야 이해하게 되더라구요.

# git flow 시작

brew install git-flow

# git flow 개념

Git-flow에는 5가지 종류의 브랜치로 나누어지게 됩니다.
항상 유지되는 메인 브랜치들(master, develop)과 일정 기간 동안만 유지되는 보조 브랜치들(feature, release, hotfix)이 있습니다.

  • master : 제품으로 출시될 수 있는 브랜치
  • develop : 다음 출시 버전을 개발하는 브랜치
  • feature : 기능을 개발하는 브랜치
  • release : 이번 출시 버전을 준비하는 브랜치
  • hotfix : 출시 버전에서 발생한 버그를 수정 하는 브랜치
